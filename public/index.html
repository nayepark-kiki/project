물류용 스마트글라스 화면을
빠르게 설계하기 위한 공통 기준

기획/디자이너는 화면 판단 기준을 훑고, 개발자는 토큰과 컴포넌트 값을 바로 가져갈 수 있도록
정리한 실무형 디자인 시스템 페이지입니다.

Design.md 다운로드

같은 원칙을 다른 깊이로 보기

기획/디자이너용

화면을 결정할 때 확인

  • 이 화면이 작업 지시인지, 선택인지, 완료 상태인지 정의
  • 카메라 배경이 필요한지 먼저 판단
  • 표시 필드와 필드 우선순위 확정
  • 발화가 보조인지 필수 인터랙션인지 결정
  • 고객사별로 바뀌는 용어, 순서, 색상 분리
개발자용

화면을 구현할 때 확인

  • 안전 여백: 상단 80px, 하단 15px, 좌우 20px 이상
  • 토큰: 색상, 폰트, 간격, 상태값 복사
  • 컴포넌트: 작업 지시 필, 대상 데이터 카드, 발화 명령 칩, 리스트 항목
  • 상태 색상: 성공 초록, 오류 빨강, 선택 파랑
  • 선택 UI: 카메라 OFF, 검은 배경, 발화 명령 표시

공통 원칙

피킹과 재고조사뿐 아니라 모든 물류용 스마트글라스 화면에 적용하는 기본 판단 기준입니다.

01

하단 중앙 우선

주요 작업 정보는 하단 중앙에 배치합니다. 안전모 착용 시 하단 영역이 더 안정적으로 보일 수 있습니다.

02

중앙 상시 점유 지양

중앙은 스캔 결과, 위험 경고, 전체 완료, 사용자 확인이 필요한 상태에 제한적으로 사용합니다.

03

최대 3줄

상시 노출되는 작업 정보는 1줄 핵심과 1줄 보조를 권장하며, 최대 3줄을 넘기지 않습니다.

04

선택 화면은 카메라 OFF

리스트, 카테고리 선택, 이슈 보고는 검은 배경 선택 UI로 구성합니다.

05

필드 임의 고정 금지

BIN, SKU, 모델명, 수량, 진행률은 예시 필드입니다. 실제 표시 정보는 프로젝트별로 정의합니다.

복사 가능한 기본 값

컬러, 폰트, 여백 값은 우측 복사 버튼으로 바로 가져갈 수 있습니다.
프로젝트별 브랜드 컬러는 변경 가능하지만 상태 색상은 일관성을 우선합니다.

색상 토큰

토큰 사용처 값
color/action/primary 진행, 선택, 주요 액션 #1473ff
color/state/success 스캔 성공, 완료 #12b52a
color/state/error 불일치, 오류, 중단 #f04438
color/state/issue 이슈, 주의, 확인 필요 카드 #d8b4b7
color/background/black 선택 UI, 상태 화면의 카메라 OFF 배경 #000000

타이포그래피 토큰

토큰 사용처 값
font/family/base 기본 UI 폰트 Pretendard
font/hud/command 하단 주요 작업 지시 24px / 700
font/hud/data BIN, 모델, 수량 등 주요 데이터 30px / 800
font/choice/label 선택 UI 라벨 20px / 700
font/caption/status 진행률, 보조 상태 13px / 700

안전 여백과 간격

토큰 사용처 값
safe/top 모든 화면 상단 여백 80px
safe/bottom 모든 화면 하단 여백 15px
safe/side 모든 화면 좌우 여백 20px
radius/control 카드, 패널, 선택 UI 8px

화면 영역과 배치

주요 작업 정보는 하단 중앙을 우선합니다.
상단과 좌우는 보조 정보 영역이며, 중앙은 즉시 인지가 필요한 상태에만 제한적으로 사용합니다.
